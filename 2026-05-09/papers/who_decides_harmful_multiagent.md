## 方法概述 (中文)

现有内容审核系统采用"一刀切"的统一有害内容判定策略，忽视了不同用户群体、不同文化背景对内容敏感度的差异，导致大量误判（过度审核或审核不足）。

本文提出 **基于 LLM 的多智能体个性化推理框架**，将内容审核决策拆分为三个协作角色：
1. **领域专家 Agent（Domain Expert Agents）**：针对特定有害内容领域（如仇恨言论、自伤、性暗示、虚假信息等）进行专项判断。
2. **管理者 Agent（Manager Agent）**：汇总各专家 Agent 的判断，协调裁决。
3. **幽灵档案 Agent（Ghost Profile Agent）**：构建用户的隐性敏感度档案（Ghost Profile），根据用户的历史行为、偏好、人口学特征等推断其内容容忍阈值，实现个性化审核策略。

框架的核心创新是将"谁来决定什么是有害的"这一问题从单一平台层面下移至用户个性化层面，同时保持平台政策基线约束。实验显示相比非个性化基线准确率提升最高 **32%**。

## 故事线 (Story Arc)

> **现状不足：** 传统内容审核系统以平台统一标准裁决"有害性"，忽略用户个体差异，导致过度审核（误杀合规内容）或审核不足（漏掉对特定群体有害的内容）。
>
> **我们的解法：** 用多 Agent 框架将审核决策分解为专家判断 + 个性化档案推理 + 管理者协调，根据用户画像动态调整审核阈值，实现既遵守平台政策又契合用户需求的个性化内容治理——Multi-Agent Personalised Moderation。

## 创新点分析

| 维度 | 描述 |
|------|------|
| 核心创新 | Ghost Profile Agent 基于隐性用户档案实现个性化审核阈值推断，突破"一刀切"困境 |
| vs. 先前工作 | 先前工作多用统一分类器或简单 prompt 工程；本文首次系统性引入用户个性化档案到内容审核决策链 |
| 多 Agent 协同 | 领域专家+管理者+幽灵档案三层结构，职责清晰，可扩展到新有害类别 |
| 可行性 | LLM-based，无需重新训练专有模型，可部署于现有平台 |
| 局限 | 用户档案构建的冷启动问题；个性化可能被滥用为逃避管控的借口；多 Agent 调用开销较大 |
