## 方法概述 / Method Overview

Latent Memory 解决多模态 RAG 中上下文 token 成本高的核心问题：把检索到的多模态证据（文本+图像）**压缩为少量 latent token**，再交给生成模型使用。其核心是一个可训练的压缩器（Latent Compressor），把长证据序列映射到固定长度的 latent 表示，并在下游任务上端到端优化。

**Story arc:** 多模态 RAG 往往需要把多段文本和多张图片拼接进上下文，推理成本随证据长度线性上升，难以规模化 → Latent Memory 通过学习式压缩，把证据先"变成少量 latent 记忆"再喂给生成器，实现更低成本的多模态证据利用，且效果不降甚至略升。

---

## 创新点 / Innovation

1. **面向多模态证据的 latent 压缩记忆** — 端到端可训练压缩器
2. **固定长度 latent token 作为 RAG 的"可控上下文接口"** — 解耦证据长度与上下文开销
3. **跨域与多任务验证 token 效率优势** — OOD 数据集上优于非压缩方案
4. 适用于内容审核/检索理解等需要大量证据的场景
