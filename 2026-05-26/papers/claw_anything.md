## 方法概述 (中文)

当前 AI Agent 评估基准（如 WebArena、OSWorld、GAIA 等）仅为 Agent 提供**有限的用户状态切片**：单任务会话历史、简单后端接口、单设备 GUI。现实中的"永远在线（Always-On）"个人助理需要跨越月度级活动历史、多个相互依赖的后端服务和跨设备（PC+手机+IoT）的 GUI+CLI 混合交互——这种广泛上下文下的能力完全未被现有基准覆盖。

**Claw-Anything** 提出沿三个维度**扩展 Agent 上下文**：
1. **长视野活动历史（Long-horizon activity histories）**：通过多轮事件注入（multi-round event injection）模拟数月用户活动，生成复杂的世界状态，包含不相关事件和冲突信号以产生现实噪声。
2. **相互依赖的后端服务（Interdependent backend services）**：跨日历、邮件、购物记录、地图等多后端服务的状态依赖链路。
3. **GUI + CLI 跨设备交互（Integrated GUI and CLI across multiple devices）**：同时要求 Agent 在手机 APP、桌面 GUI 和命令行之间协调操作。

---

## 故事线 (Story Arc)

> **现状不足：** 现有 Agent 基准提供的用户状态过于简单，无法捕捉真实"永远在线"助理场景中的广度、噪声与跨域依赖，导致评测结果在实际部署中严重失效。
>
> **我们的解法（Claw-Anything）：** 通过三维度上下文扩展 + 事件注入噪声模拟，构建更真实的长视野个人助理基准，推动 Agent 上下文扩展研究。

## 创新点分析

| 维度 | 描述 |
|------|------|
| 核心创新 | 三维度 Agent 上下文扩展（时间深度、服务广度、交互跨度）同时覆盖；事件注入生成现实噪声 |
| vs. 先前工作 | 已有基准（GAIA、OSWorld、WebArena）仅覆盖有限状态空间；Claw-Anything 显著扩大上下文规模 |
| 可行性 | 基准工程挑战较大，但方法论清晰；事件注入模拟是已验证的合成数据范式 |
| 局限 | 仅基准论文，无配套高性能 Agent 解决方案；长视野噪声设计的覆盖性有待社区验证 |
