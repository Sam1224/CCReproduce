## 方法概述 (中文)

直播是当前增速最快的内容电商形式。与短视频不同，直播内容实时变化（主播状态、互动话题、商品展示动态切换），导致现有推荐算法面临：

1. **计算资源利用不足**：现有算法 FLOPs 偏低，限制了性能进一步提升的上限；
2. **静态内容表征**：传统方法将直播间视为静态 ID，无法捕捉直播内容随时间变化的动态特征；
3. **多模态信息融合不充分**：主播视频、语音、弹幕等多模态信号利用不足。

**SSRLive** 提出静态+动态语义 ID 统一框架：

1. **生成模块（Generative Module, 编码器-解码器架构）**：
   - **静态 SID（Static Semantic ID）**：基于直播间固定特征（主播 ID、历史风格、类目）生成，捕捉稳定的内容风格表征；
   - **动态 SID（Dynamic Semantic ID）**：实时融合直播过程中的多模态特征（视频帧、音频、弹幕文本），生成随时间更新的动态内容表征；二者融合提供完整的时序语义覆盖。

2. **判别模块（Discriminative Module）**：将 SID 与用户特征结合，借助用户-主播互动数据（关注、礼物、评论）做特征增强，执行多任务预测（点击、关注、GMV 等）。

两模块在统一架构中联合训练，兼顾生成式语义表征质量与判别式推荐精度。

---

## 故事线 (Story Arc)

> **现状不足：** 直播推荐把直播间当静态 ID，无法捕捉内容实时变化；多模态直播信号利用不足，FLOPs 低、性能受限。
>
> **我们的解法：** SSRLive 用生成式 SID 统一静态风格+动态内容（多模态实时更新），再通过判别式模块融合用户互动数据做多任务预测——生成-判别双模块协同，让直播推荐"能理解内容变化的内容"。

## 创新点分析

| 维度 | 描述 |
|------|------|
| 核心创新 | 静态+动态 SID 双轨生成，首次系统性地为直播内容设计时序语义 ID |
| 多模态融合 | 视频+音频+弹幕多模态信号统一编码进动态 SID |
| vs. 先前工作 | 先前生成式推荐（如 TIGER/GRec）专注商品 ID；SSRLive 将语义 ID 扩展至动态直播场景 |
| 工业验证 | 淘宝天猫线上 A/B：GMV +0.72%，观看时长 +3.38% |
| 局限 | 动态 SID 实时更新对推理延迟有要求；仅在阿里巴巴生态验证 |
