# AIR: Atomic Intent Reasoning for E-Commerce Recommendation

Toy reproduction of **"AIR: Atomic Intent Reasoning for E-Commerce Cross-Domain Recommendation"**  
arXiv: 2606.10357 | KDD 2026 | Score: 83/100

## Core Innovation

Standard behavior sequences are noisy: most clicks are exploratory and don't reveal true purchase intent.  
AIR compresses the full interaction history into **Atomic Intent Units (AIUs)** — concise, high-signal natural-language descriptions of what the user actually wants to buy.

Three-stage pipeline:
1. **AIU Extraction** (offline): compress noisy behavior sequence → 1-3 atomic intent sentences per user
2. **Offline LLM Reasoning** (offline): LLM reads AIUs and reasons about user's cross-domain purchase intent
3. **Online Knowledge Distillation** (online): lightweight student model distills LLM reasoning for real-time ranking

```
User History (20+ items, noisy)
        │
        ▼
[AIU Extractor] ──── Atomic Intent Units (3 sentences)
        │
        ▼
[LLM Reasoner] ──── Cross-domain Intent Embedding (D=256)
        │
        ▼
[Online KD Module] ──── Student embedding → Ranker
```

## Quick Start

```bash
# 1. Generate toy data
cd AIR
python data/toy_dataset.py

# 2. Run training (SFT → KD)
python train.py --stage both --batch_size 4

# 3. Evaluate
python evaluate.py --test_data data/toy_data/test.json
```

## Key Design Choices

| Component | Taiji (baseline) | AIR |
|-----------|-----------------|-----|
| Behavior input | Raw sequence | Atomic Intent Units |
| LLM role | CoT reasoning | Cross-domain intent synthesis |
| Online serving | Intent embedding | Distilled student model |
| Cross-domain | Same domain only | Content → E-commerce |
