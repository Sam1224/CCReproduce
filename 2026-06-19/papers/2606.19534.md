## 方法概述 / Method Overview

**中文**

PerceptionDLM 是面向高效并行区域感知的多模态扩散语言模型。第一阶段构建 PerceptionDLM-Base（SigLIP-2 视觉编码器 + LLaDA-8B 扩散语言骨干，经 4 阶段视觉指令微调），成为目前最强的开源扩散 VLM 基础模型。在其之上引入三个核心组件：(1) **区域提示**（为每个掩码区域配一个可学习嵌入，绑定区域身份）；(2) **RoI 对齐特征回放**（将每个区域的 RoI 池化特征重新注入扩散解噪流）；(3) **结构化块状注意力掩码**（强制每个区域的 token 只关注全局视觉 token、共享文本提示、自身 RoI 特征与自身描述 span）。三者协同，使模型在一次扩散去噪过程中同时为多个掩码区域生成描述，区域间严格独立，彻底消除自回归区域描述的串行延迟瓶颈。此外，作者构建了同时评测质量与效率的 **ParaDLC-Bench** 基准（Parallel Dense Local Captioning Bench）。

**English**

PerceptionDLM is a multimodal diffusion language model for efficient parallel region perception. Stage 1 builds PerceptionDLM-Base (SigLIP-2 vision encoder + LLaDA-8B diffusion backbone, 4-stage visual instruction tuning), the strongest open diffusion VLM baseline. Three novel components are then added: (1) **Region Prompting** — a learnable embedding per masked region binds region identity; (2) **RoI-Aligned Feature Replay** — each region's pooled RoI features are injected back into the denoising stream; (3) **Structured (Block-Wise) Attention Masks** — each region's tokens attend only to global visual tokens, the shared text prompt, their own RoI features, and their own caption span, enforcing strict inter-region independence. Together, they allow the model to simultaneously denoise captions for all masked regions in a **single diffusion pass**, eliminating the linear serial-latency bottleneck of autoregressive region captioning. A new benchmark **ParaDLC-Bench** jointly measures accuracy and efficiency.

## 故事弧线 / Story Arc

> **现有方法不足 → 我们设计了 Y 来解决它**

现有 MLLM（如 GPT-4o、Qwen2-VL）大多依赖自回归（AR）解码：区域越多，延迟线性暴增。扩散语言模型（如 LLaDA）天生并行，但直接用于多区域细粒度描述时，存在区域感知弱、多区域 token 相互串扰两大问题。PerceptionDLM 通过"区域提示绑定身份 + 结构化掩码隔离区域 + RoI 特征增强感知"三管齐下，把多区域描述从串行变为真正并行，并贡献首个同时衡量质量与效率的 ParaDLC-Bench 来支撑评测。

## 创新性分析 / Innovation

与先前工作的**核心差异**：

1. **首次将扩散 LM 的 token 级并行性用于并行区域感知**：以往扩散 VLM 用于整图生成/描述，未系统研究多区域细粒度并行 captioning。
2. **三组件协同设计**：区域提示（身份绑定）+ 结构化注意力掩码（隔离） + RoI 特征回放（感知增强）形成闭环，三者缺一则退化为单区域或串行。
3. **ParaDLC-Bench**：首个同时评估并行区域描述质量与推理效率的基准，填补评测空白。
4. **新颖性合理可行**：扩散 LM 并行性是真实的硬件优势，结构化掩码无额外参数，设计简洁。
