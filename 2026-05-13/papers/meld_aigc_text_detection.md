## 方法概述 (中文)

随着 LLM 深度融入写作场景，**AI 生成文本检测**对学术诚信、内容审核和来源追踪至关重要。实际部署中，检测器不仅需要高聚合 AUROC，还需满足：①对攻击/改写的鲁棒性；②对新型生成器和领域的迁移能力；③低假阳率（FPR）。

**MELD** 在二元检测基础上增加**辅助监督头**，将以下任务附着在共享编码器上：
- **生成器家族分类**（GPT-4 / Claude / Llama 等来源识别）
- **攻击类型识别**（paraphrasing / watermark removal 等）
- **源域分类**（新闻 / 代码 / 学术 / 社交媒体等）

四个损失函数通过**同方差不确定性加权（Homoscedastic Uncertainty Weighting）**自动平衡，避免手动调权。

---

## 故事线 (Story Arc)

> **现状不足：** 现有 AIGC 检测器在攻击鲁棒性、跨域迁移、低 FPR 三个维度不能同时满足；单纯二元分类丢失了大量可供利用的结构信息（来源、攻击类型、域）。
>
> **我们的解法：** MELD 利用辅助任务提供额外结构化监督，同时用不确定性加权自动平衡多任务损失，提升主任务检测性能并增强泛化能力。

| 维度 | 描述 |
|------|------|
| 核心创新 | 同方差不确定性加权的多任务学习框架，将生成器家族/攻击类型/源域作为辅助任务 |
| vs. 先前工作 | DetectGPT、Ghostbuster 等均为单任务检测；本文的辅助任务信号提供结构化约束 |
| 可部署性 | 定位为"可部署的生产级检测器" |
| 局限 | 对中文/多语言 LLM 生成文本的检测能力未充分评估 |
