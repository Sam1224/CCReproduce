## 方法概述 (中文)

电商搜索相关性（Query-Product Relevance）是用户体验的基石。现有优化管线依赖**多个人类角色**协作：用户反馈差案例 → 产品经理定义标准 → 标注员标注数据 → 算法工程师优化模型 → 评估者评估效果。这条流程耗时、高成本、且人为偏差难以消除。

**案例驱动多智能体框架（Case-Driven Multi-Agent Framework）** 把上述角色全部替换为 LLM 智能体：

1. **User Agent（用户智能体）：** 模拟用户对话行为，自动发现搜索差案例（bad cases），无需真实用户举报。
2. **Annotator Agent（标注智能体）：** 对差案例进行多路推理（multi-path reasoning）后生成相关性标签，产出"生成式相关性模型（GRM）"的训练数据。论文报告标注精度比人工提升 **2.4%**，成本降低 **75.4%**。
3. **Optimizer Agent（优化智能体）：** 自主分析差案例的根因（policy gap / data gap / model gap），并自动触发相应的修复动作——更新策略文档、生成合成训练数据或调整模型超参。

三智能体形成**闭环迭代**系统，可持续自我进化，无需人工介入。

## 故事线 (Story Arc)

> **现状不足：** 电商搜索相关性优化依赖昂贵的人工标注和碎片化的多角色协作流程，差案例识别滞后、策略更新慢、标注成本高。
>
> **我们的解法：** 以"差案例驱动"为引擎，用三个 LLM 智能体（User Agent / Annotator Agent / Optimizer Agent）自动化整条相关性优化管线，形成自主演化的闭环系统——从差案例发现到根因分析，再到模型修复，一气呵成。
