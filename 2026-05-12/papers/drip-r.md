## 方法概述 (中文)

LLM Agent 被越来越多地部署在零售/电商场景中执行常规但后果重要的任务（退款处理、价格核查、库存决策、投诉处理），这些任务受领域策略约束。然而，现实中的业务策略天然存在歧义——同一条政策在不同情境下可能有多种合理解读。

**现有评测的问题**：当前 Agent 基准（GAIA、WebArena、ToolBench 等）均假设任务策略清晰、无歧义，缺少对"政策模糊场景下 Agent 决策能力"的专项评测。

**DRIP-R 的构建方法：**

1. **政策来源**：从真实零售商的公开政策文件（退款政策、会员权益、促销条款等）中识别固有的政策歧义点。
2. **歧义场景构建**：系统性地利用这些歧义点构造"无单一正确答案"的决策场景，每个场景存在至少两种合理且可辩护的解读路径。
3. **评估维度**：
   - 决策质量：Agent 是否做出一致、可辩护的决策
   - 推理质量：Agent 是否正确识别歧义所在并明确表述
   - 一致性：面对等价的歧义表述时，Agent 是否保持一致
4. **多 Agent 测试**：在基准上评测主流 LLM（GPT-4o、Claude、LLaMA 等）的表现。

## 故事线 (Story Arc)

> **现状不足：** 零售/电商 Agent 的部署依赖业务策略，而策略在现实中永远是"不完整的"——没有团队能预见所有边界情况。现有基准假设策略清晰，无法测试 Agent 在政策空白/歧义下的判断能力，这正是生产中最容易出错的场景。
>
> **我们的解法：** DRIP-R 从真实零售政策中系统挖掘歧义点，构建无单一正确答案的决策任务，评测 LLM 面对政策空白时的推理策略、一致性和歧义识别能力。

## 电商应用场景

- **平台规则解释 Agent**：处理商家投诉时，Agent 需要理解平台规则的边界情况和歧义解读
- **智能客服**：在退款/维权场景中，政策歧义处理是核心能力
- **达人/内容创作者治理**：平台内容政策的歧义解读是治理难点之一，此评测方法可延伸至内容合规场景
